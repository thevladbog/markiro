Логотип

02 / 08

Знак — модульная сетка DataMatrix, в которой читается буква «М». Один модуль — зелёный: удачный скан. Знак и шрифтовая часть маркиро (Plex Mono, строчные) живут вместе и порознь.

Компакт-знак: app icon · шапка приложения · favicon 16px
Охранное поле: ширина одного модуля знака со всех сторон.
Минимум: знак 16px, логотип с текстом — 96px по ширине.
В один цвет зелёный модуль становится цветом печати — форма «М» несёт знак без цвета.
Кириллица/латиница: маркиро / markiro — равноправные написания, всегда строчными.

Иконки

05 / 08

Сетка 24×24, штрих 2px, прямые углы и обрубленные концы — характер «прибора». Скругления не используются. Заливка допустима только у статусных знаков.

Правила
  • Штрих 2px при 24px; масштабируется пропорционально.
  • stroke-linecap: butt, углы без радиуса.
  • Статусная иконка всегда уникальна по форме: ✓ ✕ ⧉ ⟳ — цвет дублирует, не заменяет.
  • В цехе минимальный размер 32px, в офисе — 16px.
Нельзя
  • Скругления, «дружелюбные» закруглённые концы штриха.
  • Заливка обычных иконок, градиенты, тени.
  • Смешение с иконками других библиотек без адаптации к сетке.
  • Иконка без подписи для нестандартного действия.

Тон голоса

06 / 08

Мы говорим с производственниками, не с ИТ-отделом. Просто, по делу, конкретно. Всегда: что случилось → что делать. Русский — первый язык; английский держит тот же регистр.

Так
«Линия работает. Отчёт готов за две минуты.»
«Принтер не отвечает. Проверьте кабель и нажмите „Повторить“.»
«Коды кончатся через ~40 минут. Заказать ещё?»
«Нет связи с Честным ЗНАКОМ. Работаем офлайн, отправим позже.»
Не так
«Цифровая трансформация бизнес-процессов маркировки»
«Ошибка 0x80070057: недопустимый параметр»
«Упс! Что-то пошло не так 🙈»
«Осуществляется синхронизация данных с ГИС МТ»
Правила: глагол вместо существительного («заказать», не «осуществить заказ») · числа цифрами · без эмодзи и восклицаний · ошибка всегда заканчивается действием пользователя · «вы» строчными.

Аудио-сигналы станции

07 / 08

Короткие прямые тоны (square), различимые в шумном цехе без взгляда на экран. Различие — по количеству нот и регистру, не только по высоте: успех — один высокий, ошибка — два низких длинных, дубликат — два средних коротких, короб — восходящая триоль.

✓ Успех
1 нота · 1175 Гц (D6) · 70 мс
самый частый звук — максимально короткий
✕ Ошибка
2 ноты · 220 Гц (A3) · 160+160 мс
низкий регистр — не спутать с успехом даже в шуме
⧉ Дубликат
2 ноты · 740 Гц (F#5) · 60+60 мс
средний регистр: «внимание», не «стоп»
▣ Короб собран
3 ноты · 660→880→1175 Гц · 70+70+120 мс
восходящая триоль — «этап закрыт»
Правила: громкость настраивается на станции · успех можно отключить, ошибку — нельзя · никакой музыки и мелодий · пауза между повторами ошибки ≥ 2 с, чтобы сигнал не сливался в фон.

Do / Don't

08 / 08
Do
  • Знак на чистом поле, монохром для документов и печати.
  • Зелёный модуль — единственный акцент знака.
  • Статус всегда = цвет + форма + текст.
  • Счётчики и коды — Plex Mono, tabular-nums.
  • Логотип строчными: маркиро / markiro.
  • Тёмная тема — родная для цеха, светлая — для офиса.
Don't
  • Не перекрашивать модули знака и не «оживлять» их анимацией в статике.
  • Не ставить знак на пёстрые фото без плашки.
  • Не использовать статусные цвета в декоре, кнопках навигации, графиках без статусного смысла.
  • Не набирать логотип другим шрифтом и не писать «МАРКИРО» капсом.
  • Не вводить дополнительные акцентные цвета.
  • Не растягивать и не наклонять знак.
