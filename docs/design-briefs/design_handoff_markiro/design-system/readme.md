# Маркиро — дизайн-система

Платформа маркировки «Честный ЗНАК» для производственных линий (пиво, вода, безалкогольные; завтра — молоко, БАДы, одежда). Бренд-направление «Прибор» (бриф 01, утверждено): инженерный монохром, цвет зарезервирован за статусами. Источники: папка `q/docs/design-briefs/` (брифы 00–05), утверждённые артефакты бренда в корне проекта (`Маркиро — мини-брендбук.dc.html`).

## Одна система, два режима

- **Офис** (админ-панель, лендинг): светлая тема, плотность обычного SaaS, контролы 40px, текст 14–16px. Ориентир: Linear/Stripe.
- **Цех** (станция линии): тёмная тема — родная, контролы 64px (перчатка), текст от 18px, счётчики 48–96px, читаемость с 1,5–2 м. Индустриальный HMI со вкусом.

Оба режима — общие токены, иконки, семантические цвета и ДНК компонентов: цеховая кнопка — это офисная, укрупнённая, а не другой продукт. Переключение темы — атрибут `data-theme="dark"` на контейнере.

## CONTENT FUNDAMENTALS

Говорим с производственниками, не с ИТ. Просто, по делу: что случилось → что делать.

- «Принтер не отвечает. Проверьте кабель и нажмите „Повторить“.» — не «Ошибка 0x80070057».
- «Коды кончатся через ~40 минут. Заказать ещё?» — не «Осуществляется резервирование КМ».
- Глагол вместо отглагольного существительного; числа цифрами; без эмодзи и восклицаний; «вы» строчными; ошибка всегда заканчивается действием.
- Русский — первый язык. RU-строки длиннее EN в 1,3–1,5× — компоненты обязаны их вмещать, статусные слова не сокращаются («Синхронизация», не «Синхр.»).

## VISUAL FOUNDATIONS

- **Цвет:** бренд-монохром (чернила #17161A, бумага #FAFAF8, панель #F0EFEA). Единственный акцент — зелёный модуль знака (#0FAF56 свет / #3DDC7A тьма). Семантические статусы — первоклассные граждане: OK-зелёный, ошибка-красный, дубликат-янтарный, синхронизация-синий; каждый определён для обеих тем с контрастом ≥6.8:1 (свет) и ≥6:1 (тьма); в цехе статус = весь интерфейс. Статус никогда не только цветом: цвет + форма-иконка + текст.
- **Типографика:** IBM Plex Sans (интерфейс) + IBM Plex Mono (коды, счётчики, логотип). Полная кириллица. tabular-nums везде, где числа. Офисная шкала 12–32, цеховая 18–96.
- **Формы:** прямые углы в графике (иконки, ячейки box-fill радиус 2), у контейнеров радиусы 4/8/12. Границы важнее теней; тени слабые (--shadow-1..3) только для всплывающих слоёв.
- **Анимация:** утилитарная и короткая — width/opacity 150–300ms, спиннер, шиммер скелетона; никаких пружин и подпрыгиваний. Сигнальные вспышки цеха: OK ~400 мс, ошибка — до подтверждения.
- **Hover** (только офис): brightness(0.92); pressed: translateY(1px) + brightness(0.85). Hover никогда не единственная аффорданса — цех touch-first.
- **Фокус:** видимое кольцо 2px (--focus-ring, синее) для клавиатуры в админке.
- **Фоны:** плоские заливки, без градиентов, изображений и текстур.

## ICONOGRAPHY

Собственный набор (~45 знаков) в `components/icons/Icon.jsx`: сетка 24×24, штрих 2px, прямые углы, butt caps, без заливки (кроме модульных мини-квадратов). Категории: навигация, статусы (ok-square, error-square, duplicate, sync, offline, alert), оборудование (printer, scan, agent, camera), агрегация (unit, box, pallet, disassemble), смена/люди. Эмодзи не используются. В цехе иконки ≥32px и всегда с подписью. Не смешивать со сторонними библиотеками без адаптации к сетке.

## Аудио-сигналы (цех)

Прямые square-тоны: успех 1×1175 Гц 70 мс; ошибка 2×220 Гц 160 мс; дубликат 2×740 Гц 60 мс; короб 660→880→1175 Гц. Спецификация и прослушивание — в мини-брендбуке (стр. 07). Ошибку отключить нельзя, успех — можно.

## Индекс

- `styles.css` → `tokens/` (fonts, colors, typography, spacing)
- `assets/` — logo.svg, logo-dark.svg, mark.svg, mark-mono.svg
- `components/icons/` — Icon (+ реестр ICONS)
- `components/forms/` — Button, Input (в т.ч. type="date"), Select, PinPad
- `components/display/` — Card, StatusChip, Tabs, Breadcrumbs, Table
- `components/feedback/` — Alert+Toast, Modal+FullScreenDialog, SignalOverlay, Progress (ProgressBar+RingCounter), BoxFill, EmptyState+Skeleton
- `components/navigation/` — Sidebar, TaskSwitcher, StatusBar
- `guidelines/` — карточки фундамента (цвета, тип, отступы, бренд, тон)
- UI-киты админки и станции — брифы 03–04, следующие этапы.

## Intentional additions

- `Icon` — обёртка фирменного набора (нужна всем компонентам).
- `EmptyState`/`Skeleton` — паттерны состояний из брифа 02 оформлены компонентами.

## Каверзы

- Шрифты подключены с Google Fonts (IBM Plex Sans/Mono, кириллица); локальные бинарники не привозились.
- Карточки компонентов в Design System — статические HTML-специмены (визуально зеркалят JSX); источник правды — сами .jsx.
