# Handoff: Маркиро — платформа маркировки «Честный ЗНАК»

## Overview

Маркиро — SaaS для производственных линий (пиво, вода, безалкогольные; платформа универсальна для групп Честного ЗНАКА): сериализация, валидация и агрегация кодов маркировки. Пакет покрывает три приложения на общем бренде и дизайн-системе:

1. **Админ-панель** (`prototypes/admin-panel.dc.html`) — офисный веб-апп менеджера: смены, каталог, коды, этикетки, экспорт, пользователи.
2. **Станция линии** (`prototypes/line-station.dc.html`) — touch-first приложение оператора у конвейера (планшет 1280×800, POS 1024×768 / 1280×1024), тёмная тема по умолчанию.
3. **Лендинг** (`prototypes/landing.dc.html`) — маркетинговая страница, конверсия в заявку на демо.

Плюс `prototypes/brandbook.dc.html` — утверждённый мини-брендбук (направление «Прибор»), и `design-system/` — токены, компоненты, иконки.

## About the Design Files

Файлы в пакете — **дизайн-референсы в HTML** (прототипы, показывающие внешний вид и поведение), а не production-код. Задача — **воссоздать эти дизайны в целевом стеке** (React/Vue/и т.п.) по паттернам вашей кодовой базы; если стека ещё нет — выбрать подходящий фреймворк и реализовать там. `.dc.html` файлы открываются в браузере напрямую (шаблон внутри `<x-dc>` + класс логики; `support.js` — рантайм прототипа, в production не переносится). Читайте их как источник разметки, инлайн-стилей и поведения.

`design-system/components/*/*.jsx` — эталонные React-компоненты с инлайн-стилями на CSS-переменных: их можно переносить в кодовую базу почти как есть (заменив `.d.ts`-контракты на props вашего стека). `*.prompt.md` — краткая документация каждого компонента, `*.card.html` — статические специмены.

## Fidelity

**High-fidelity.** Цвета, типографика, отступы, размеры контролов и тексты — финальные, воспроизводить точно. Данные в таблицах/счётчиках — демо-заглушки.

## Design Tokens

Источник правды: `design-system/tokens/*.css` (подключаются через `design-system/styles.css`). Ключевое:

- **Шрифты:** IBM Plex Sans (интерфейс) + IBM Plex Mono (коды, счётчики, логотип), полная кириллица, Google Fonts. Все числа: `font-variant-numeric: tabular-nums`.
- **Бренд-монохром:** чернила `#17161A`, бумага `#FAFAF8`, панель `#F0EFEA`, линии `#E0DED7`/`#C9C6BD`. Единственный акцент — зелёный `#0FAF56` (свет) / `#3DDC7A` (тьма).
- **Семантические статусы** (первоклассные граждане бренда, всегда цвет + форма-иконка + текст):
  - OK: `#116335` на `#E7F6EC` (свет) / `#3DDC7A` на `#142E1D` (тьма)
  - Ошибка: `#A1231A` на `#FDEAE7` / `#FF6B5E` на `#391A16`
  - Дубликат: `#8A4C07` на `#FDF0DC` / `#FFB84D` на `#33250E`
  - Синхронизация: `#1A4F9C` на `#E5EFFC` / `#6DB2FF` на `#152740`
- **Тёмная тема** — атрибут `data-theme="dark"` (полный набор переменных в `tokens/colors.css`).
- **Радиусы** 4/8/12, тени слабые (граница важнее тени), фокус-кольцо 2px `#1A4F9C`.
- **Контролы:** офис 40px (компакт 32), цех 64px (перчатка). Текст: офис 12–32px, цех минимум 18px, счётчики 48–96px.

## Screens / Views

### Админ-панель (офис, светлая тема, desktop-first 1440)
Сайдбар 224px (`#F0EFEA`, логотип, разделы с бейджами) + контент. Экраны: Обзор (KPI-карточки 32px mono-счётчики, «Смены прямо сейчас», последние экспорты), Каталог (таблица с фото-слотами 44px, статус «Черновик — заполнить»), Карточка продукта (фото 220×200, GTIN/группа/короб/паллета/шаблон; черновик блокирует запуск смены на станции), Смены (фильтры-чипы, план/факт с прогрессом), Смена (счётчики, лента сканирований, агрегация паллеты), Этикетки (список шаблонов + WYSIWYG-превью 100×100), История кода (таймлайн), Экспорт (формирование + история), Пользователи (менеджеры + операторы с PIN), Настройки (организация, ГИС МТ токен, линии). Таблицы: `overflow-x: auto`, числовые ячейки nowrap + mono.

### Станция линии (цех, тёмная тема, touch)
Экраны: 1 Вход (выбор оператора + PIN-пад 72px клавиши, альтернатива — скан бейджа); 2 Выбор смены (карточки с фото продукта, статусом, прогрессом; «+ Новая смена»); 3 Новая смена (скан продукта → режим → вместимость короба −/+ и коробов на паллете −/+; **продукта нет в каталоге → блокирующий экран «обратитесь к администратору»**, ад-хок создание запрещено); 4 Валидация (сигнал последнего скана, огромный счётчик терминала 64–96px); 5 Агрегация (в центре — сетка короба 10 колонок с заполнением ячеек, «Короб № N», «14 / 20» 56px, полоса паллеты; цифры смены/терминала/дублей — в правой панели над лентой сканов; кнопки «Закрыть короб» / «Закрыть паллету»); 6 Исключения (4 карточки, всё подтверждается сканом); 7 Рабочее место (принтер/сканер через локальный агент, громкость 4 ступени, тест звука); 8 Конфликты синхронизации (прогресс отправки, список кросс-терминальных дублей — решает менеджер, оператор только информирован; состояние «нет бумаги» с очередью этикетки).
Статус-бар 56px всегда виден: сеть/офлайн-очередь, устройства, «+2 терминала», продукт с мини-фото, оператор, часы.

### Сигнальная система (ядро UX станции)
Полноэкранные оверлеи, не тосты: Успех — зелёный `#0FAF56`, заголовок 96px, автоскрытие ~400мс; Дубль — янтарный `#DD9420`, держится до «Понятно», текст «где/когда сканировали впервые»; Ошибка/чужой ГТИН — красный `#C0392B`, до подтверждения; Короб собран / Паллета закрыта — тёмный, ~1600мс. Каждый сигнал в паре со звуком (WebAudio, square): успех 1×1175Гц 70мс; ошибка 2×220Гц 160мс (не отключается); дубль 2×740Гц 60мс; короб 660→880→1175Гц. Громкость на станции.

### Лендинг
Секции: hero (заголовок 46px, CTA «Запросить демо» + телефон, визуал станции с box-fill) → полоса цифр (100% / 8+ часов / 2 мин / 1 день) → «боль → решение» (4 пары ✕/✓) → «Один продукт — два рабочих места» (мини-моки станции и админки) → 3 шага → схема оборудования (сканер/принтер/ПК → станция с офлайн-буфером → облако → ГИС МТ/1С/WMS) → возможности (8 карточек) + тёмная вставка про мультитерминал → группы → цена (тёмная секция, «от 15 000 ₽/мес за линию», блок готов к добавлению тарифов) → FAQ (6) → форма демо (3 поля: имя, компания, телефон) → футер. Адаптив без media-queries: `flex-wrap` + `minmax(min(Npx, 100%), 1fr)`.

## Interactions & Behavior

- Hover только в офисе (`brightness(0.92)`; pressed `translateY(1px)`), в цехе hover не носитель смысла.
- Переходы утилитарные: width/opacity 150–300ms; box-ячейка заполняется `background 150ms`.
- Станция: демо-кнопки «▶» имитируют сканер — в production это события реального сканера через локальный агент. Автозакрытие короба на N-й единице → печать этикетки → сигнал → следующий короб; паллета аналогично.
- Оффлайн: баннер «Работаем офлайн — сканы сохраняются…», очередь в статус-баре, после восстановления — прогресс отправки и разбор конфликтов.
- Ошибки всегда словами + действие («Проверьте кабель и нажмите „Повторить“»), тон — производственный, без ИТ-жаргона (см. брендбук, стр. 06).

## State Management (станция)

Прототип: см. класс `Component` в `line-station.dc.html`.
